# Kickoff Checklist — Abertura de projeto de site

> Executar quando o projeto é fechado, antes de qualquer trabalho de design ou dev.

---

## Contrato e escopo

- [ ] Proposta aprovada por escrito (e-mail, WhatsApp ou assinatura)
- [ ] Escopo documentado: o que inclui e o que não inclui
- [ ] Prazo definido e comunicado ao cliente
- [ ] Forma de pagamento definida: entrada + saldo, ou parcelado
- [ ] Condições de revisão definidas: quantas rodadas, qual o prazo para o cliente responder

## Projeto no Linear

- [ ] Issue criado no Linear com: nome do cliente, tipo de projeto, prazo estimado
- [ ] Pasta `clients/[nome-do-cliente]/website-project/` criada
- [ ] `01-briefing.md` preenchido com o que veio do discovery

## Coleta de conteúdo

- [ ] Lista de itens obrigatórios enviada ao cliente (`website-content-collection.md`)
- [ ] Data de entrega do conteúdo definida com o cliente
- [ ] Confirmado: o cliente tem logo em vetor?
- [ ] Confirmado: o cliente tem fotos boas do negócio?
- [ ] Se não tem fotos: plano alternativo definido (sessão, banco de imagem, design sem foto)

## Acessos

- [ ] Domínio: sabe onde está registrado e quem tem acesso
- [ ] Google Analytics: conta existente ou criar nova?
- [ ] Google Search Console: configurar após o launch
- [ ] E-mail que receberá os formulários do site

## Stack

- [ ] Stack definida com justificativa (`07-stack-decision.md`)
- [ ] Conta no Webflow criada (se Webflow) ou repositório criado (se Next.js)
- [ ] Ambiente de staging configurado

## Comunicação com o cliente

- [ ] Canal de comunicação principal definido (WhatsApp, e-mail, Linear)
- [ ] Frequência de updates combinada
- [ ] Explicado ao cliente: o que acontece em cada fase e o que ele precisa aprovar
