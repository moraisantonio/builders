# Pre-launch Checklist — Verificação final antes de ir ao ar

> Executar com o site em staging. Só fazer o go-live após todos os itens marcados.

---

## Conteúdo

- [ ] Nenhum texto placeholder em qualquer página ("Lorem ipsum", "Texto aqui", "Em breve")
- [ ] Todos os textos revisados com o cliente e aprovados
- [ ] Informações de contato corretas: telefone, e-mail, endereço, WhatsApp
- [ ] Horário de funcionamento correto (se aplicável)
- [ ] Preços corretos (se aparecerem no site)
- [ ] Ano de copyright correto no rodapé

## Funcionalidades

- [ ] Todos os formulários testados — envio funcionando e e-mail chegando
- [ ] WhatsApp abre o número correto
- [ ] Todos os links internos funcionando (nenhum 404)
- [ ] Todos os links externos funcionando e abrem em nova aba (`target="_blank"`)
- [ ] CTA principal de cada página funciona

## Visual

- [ ] Testado em Chrome, Safari e Firefox (desktop)
- [ ] Testado em iOS Safari (iPhone)
- [ ] Testado em Chrome Android
- [ ] Nenhum elemento quebrado em nenhum dos navegadores testados
- [ ] Favicon aparece na aba do browser

## SEO e técnico

- [ ] SEO checklist (`seo-checklist.md`) completo
- [ ] Performance checklist (`performance-checklist.md`) completo
- [ ] Accessibility checklist (`accessibility-checklist.md`) completo
- [ ] HTTPS ativo no domínio final
- [ ] Domínio correto apontando para o site

## Analytics

- [ ] Google Analytics instalado e recebendo dados no domínio final
- [ ] Eventos de conversão configurados e funcionando

## Aprovação

- [ ] Cliente fez revisão final no staging
- [ ] Cliente aprovou por escrito (e-mail ou mensagem registrada)
- [ ] Data e horário de go-live definidos com o cliente

---

*Após o go-live: executar `post-launch-checklist.md`*
