# Post-launch Checklist — Primeiros 7 dias após o go-live

---

## Nas primeiras 24 horas

- [ ] Confirmar que o site está no ar no domínio correto
- [ ] Confirmar HTTPS funcionando no domínio final (não só no subdomínio de staging)
- [ ] Verificar se Analytics está recebendo dados reais (não de preview)
- [ ] Verificar se formulários estão funcionando com o domínio final
- [ ] Verificar se WhatsApp abre corretamente com o domínio final
- [ ] Solicitar indexação das páginas principais no Google Search Console

## Na primeira semana

- [ ] Verificar no Search Console se houve erros de crawl
- [ ] Confirmar que o sitemap foi processado
- [ ] Testar o site em 3 dispositivos físicos diferentes (não só emulador)
- [ ] Compartilhar o link com o cliente e pedir feedback de primeira impressão
- [ ] Pedir ao cliente que peça avaliações para os primeiros clientes no Google (5 nos primeiros 30 dias)

## Documentação e encerramento de fase

- [ ] Preencher `11-post-launch-notes.md` no projeto do cliente
- [ ] Registrar credenciais de acesso (Analytics, Search Console, domínio) de forma segura
- [ ] Atualizar status do projeto no Linear
- [ ] Entregar ao cliente: link do site, instruções de edição (se Webflow), contatos de suporte

## Handoff ao cliente (quando não há retainer)

- [ ] Tutorial gravado ou escrito de como editar conteúdo (se Webflow)
- [ ] Lista de acesso entregue: domínio, hospedagem, Analytics
- [ ] Próximos passos recomendados documentados

---
