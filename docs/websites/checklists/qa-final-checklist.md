# QA Final Checklist — Revisão de qualidade completa

> Executar após o dev estar completo, antes de apresentar ao cliente para aprovação.

---

## Qualidade visual

- [ ] O site comunica o que o negócio faz em menos de 5 segundos (teste com alguém externo)
- [ ] CTA principal está óbvio sem precisar rolar
- [ ] Hierarquia visual clara no hero: headline → benefício → ação
- [ ] Nenhum elemento solto, desalinhado ou sem razão
- [ ] Espaçamento consistente entre seções
- [ ] Cores dentro da paleta aprovada
- [ ] Tipografia consistente (tamanhos, pesos, estilos conforme definido)

## Qualidade de copy

- [ ] Nenhum placeholder ("Lorem ipsum", "Texto aqui", "Em breve")
- [ ] Nenhum CTA genérico ("Saiba mais", "Clique aqui", "Enviar")
- [ ] Nenhuma frase com mais de 3 linhas no mobile
- [ ] Nenhum jargão técnico que o público não entenderia
- [ ] Textos revisados por gramática e ortografia

## Qualidade técnica

- [ ] Dev checklist (`dev-checklist.md`) completo
- [ ] Performance checklist (`performance-checklist.md`) completo
- [ ] SEO checklist (`seo-checklist.md`) completo
- [ ] Accessibility checklist (`accessibility-checklist.md`) completo

## Teste funcional

- [ ] Todos os formulários testados (envio + confirmação + recebimento)
- [ ] WhatsApp funcionando
- [ ] Todos os links funcionando
- [ ] Todos os CTAs levando ao lugar correto
- [ ] Navegação funcionando em todas as páginas
- [ ] 404 page configurada

## Teste cross-browser e cross-device

- [ ] Chrome desktop
- [ ] Safari desktop
- [ ] Firefox desktop
- [ ] iOS Safari (iPhone)
- [ ] Chrome Android
- [ ] Testado em pelo menos dois tamanhos de tela diferentes por plataforma

## Teste do "olho externo"

Mostrar o site para alguém que não conhece o projeto e pedir que responda:
- [ ] O que este negócio faz?
- [ ] Para quem?
- [ ] O que você faria agora nessa página?

Se não souber responder em 10 segundos, o site precisa de revisão antes de ir ao cliente.
