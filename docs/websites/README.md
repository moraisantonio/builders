# Websites — Camada operacional

> Documentação prática para criar sites com consistência, qualidade e menos retrabalho.
> Esta camada serve humanos. O agente de IA usa o `AGENT.md` na raiz.

---

## O que está aqui

| Arquivo | Para quê serve | Leia quando |
|---|---|---|
| `website-playbook.md` | Processo completo em visão macro | Antes de iniciar qualquer projeto |
| `website-quality-bar.md` | O que define um site "pronto" para a Builders | Antes de entregar ao cliente |
| `website-stack-decision.md` | Como decidir entre Webflow e Next.js | Na fase de proposta |
| `website-briefing-guide.md` | Como conduzir o discovery com o cliente | Antes da primeira reunião |
| `website-copy-guide.md` | Fórmulas e exemplos de copy por seção | Ao escrever textos |
| `website-design-guide.md` | Direção visual, paleta, decisões de UI | Ao definir a identidade |
| `website-seo-guide.md` | SEO técnico e on-page passo a passo | Antes do dev e antes do launch |
| `website-content-collection.md` | O que pedir ao cliente antes de começar | Após fechar o projeto |

---

## Subpastas

| Pasta | O que contém |
|---|---|
| `checklists/` | Checklists por fase: kickoff → discovery → design → dev → QA → launch → pós-launch |
| `templates/` | Documentos prontos para preencher por projeto |
| `patterns/` | Padrões de seção por função e por nicho (arquitetura, cafeterias) |
| `reference/` | Heurísticas, erros comuns, princípios de conversão |

---

## Ordem de leitura para um projeto novo

1. `website-briefing-guide.md` — antes da primeira conversa com o cliente
2. `templates/briefing-template.md` — durante ou após a reunião de discovery
3. `website-playbook.md` — visão geral do processo
4. `website-stack-decision.md` — quando tiver briefing em mãos
5. `checklists/kickoff-checklist.md` — para abrir o projeto
6. A partir daí: seguir os checklists em sequência por fase

---

## Relação com outros documentos

- **`AGENT.md`** (raiz) — documento do agente de IA. Tem o processo detalhado em 15 etapas, regras de design, copy e stack. A camada docs/websites/ complementa sem repetir.
- **`brain/design/design.md`** — identidade visual completa da Builders. Sempre consultar antes de definir direção visual.
- **`brain/services.md`** — escopo dos serviços. Confirmar o que está e o que não está incluído no projeto.
- **`clients/[nome]/website-project/`** — onde fica a documentação específica de cada projeto.
